Cubit — User Manual

Cinema 4D plugin suite for generating LEGO-style bricks and assemblies.

Two objects ship in the same plugin:

  • Cubit — a single parametric brick.
  • Cubify — converts one or more source meshes into a structurally-buildable brick assembly. Supports multi-source compositions (Union / Subtract / Intersect), MoGraph effectors, build animation, deforming-mesh binding, and per-brick color in Redshift.

Install

Supported packages: Windows and macOS (Apple Silicon) + Cinema 4D 2026 (any 2026.x point release — tested on 2026.0 and 2026.1). The download contains separate Windows and MacOS folders; use the one that matches your operating system.

  1. In Cinema 4D, open Edit → Preferences and click Open Preferences Folder... at the bottom-left. That opens your per-instance C4D folder. The path looks like:
    • Windows: %APPDATA%\Maxon\Maxon Cinema 4D 2026_<hash>\
    • macOS: ~/Library/Preferences/Maxon/Maxon Cinema 4D 2026_<hash>/
  2. Close Cinema 4D.
  3. Inside that folder, open plugins and unzip the matching platform's Cubit folder so the plugin ends up at: 
    Windows:  ...\Maxon Cinema 4D 2026_<hash>\plugins\Cubit\
macOS:    .../Maxon Cinema 4D 2026_<hash>/plugins/Cubit/
  4. Start Cinema 4D. Both objects appear under the plugin menu:
    • Cubit — single brick generator.
    • Cubify — source-mesh-to-brick assembly.

The package is self-contained — Python dependencies (numpy, scipy) are vendored for your platform. You should not need to install anything yourself.

Cubit — single-brick generator

Use this when you want a clean parametric brick (for stills, beauty shots, hero set dressing, or as the source for your own MoGraph cloner).

Adding it

Object menu → Cubit. The default is a 2×4 brick.

Parameters

  • Type — Brick (full plate-3 height) or Plate (1/3 height).
  • Width — studs along X (1–8).
  • Depth — studs along Z (1–8).
  • Height — plates tall (1–8). One plate is 3.2 mm, three plates make a full brick.
  • Quality — geometry detail:
    • Draft — ~1k tris. Fast.
    • Standard — ~5k tris. Good default for layout.
    • Hero — ~50k verts on a 2×3, full filleted edges, dense studs and tubes. Use for renders.

The mesh is generated with named polygon selections (body, studs, underside, tubes) so you can apply different materials per region.

Stud Logo (optional)

Replace the default flat stud top with your own embossed logo:

  • Use Custom Stud Logo — toggles the feature.
  • Logo Source Mesh — link any polygon object. Its silhouette is projected onto each stud top.
  • Logo Rotation — 0 / 90 / 180 / 270 degrees.
  • Logo Fill % — how much of the stud top the logo occupies.
  • Logo Height — embossed depth.
  • Logo Blend — how softly the logo edges fade into the stud surface.
  • Logo Sink — recess the logo below the stud surface (useful when you want the logo carved in rather than raised).

Cubify — mesh-to-brick assembly

Drop in one or more polygon objects as sources, and Cubify voxelizes them and tiles the result with bricks. Real-time preview, animation, MoGraph effectors, Redshift per-brick color all work on the live output. Multiple sources can be combined boolean-style (Union / Subtract / Intersect) for additive and carved compositions.

Quick start

  1. Object menu → Cubify.
  2. Parent a polygon object under the Cubify in the Object Manager — it auto-appears in the Sources list and the source mesh hides itself in the viewport (the brick output replaces it).
  3. Adjust Resolution in the Shape group until the silhouette matches what you want. Lower = chunkier bricks; higher = finer detail.
  4. Switch Brick Mesh Detail (Look tab) to Hero for a final render.

The Cubify object's parameters are organized into five tabs:

  • Object — Shape (voxelization, sizing), Sources list, Bind to Source Deformation, Tools.
  • Bricks — which brick sizes to use, plate-stack rules, surface finishing, smooth-top, per-brick variation.
  • Look — render quality, preview mode, custom stud logo, live-update.
  • Animate — build animation, smooth-top finishing animation.
  • Effectors — MoGraph effectors that drive matrix, color, and visibility.

Object tab

The Object tab holds everything that defines the assembly's overall shape and source-input model: the Shape group (voxelization + sizing), the Sources list, Bind to Source Deformation, and the Tools action buttons.

Shape group

Two-column layout: voxelization parameters on the left, build options + rebuild controls on the right.

  • Algorithm — voxelization backend:
    • Simple — internal voxelizer. Faster, less precise on thin features.
    • Detailed — Cinema 4D Volume (MeshToVolume) sampling. Slower but handles thin walls and concave surfaces much better. Recommended for most production work.
  • Resolution — slider 0.1–1.0 controlling voxel density. Lower = chunkier bricks, faster builds. Higher = finer detail, slower. Greys out when Use Custom Scale is on (the two are mutually exclusive sizing controls).
  • Use Custom Scale + Stud Size — when on, you set the absolute physical stud size in scene units instead of letting Cubify derive it from the source bounds. Useful when the brickified output needs to match a specific real-world scale. Stud Size greys out when Custom Scale is off.
  • Build Type — Solid (fill the whole interior with bricks) or Shell (only place bricks within Wall Thickness of the surface — lighter, hollow inside).
  • Wall Thickness — only meaningful for Shell. Number of brick layers inward from the surface.
  • Clean Small Details — removes stray single-brick protrusions that read as noise rather than detail (0–4 levels of cleanup).
  • Keep Tiny Gaps — preserves narrow gaps in the source by allowing smaller bricks; off by default for cleaner output.
  • Make Physically Accurate — when ON, the assembly must be a single connected structure (drops floating clusters; you can build it with real bricks). When OFF, Cubify prioritizes silhouette accuracy and fills boundary cells with 1×1s as needed; physically separate islands are kept.
  • Rebuild Now + Live Update — when Live Update is on (default), the assembly rebuilds after every parameter change. When off, sliders only update cached state and Rebuild Now commits. Useful on heavy scenes when scrubbing feels sluggish.

Sources list

Cubify accepts multiple source meshes and combines them with boolean-style modes. The Sources list is rendered as a Volume-Builder-style inset panel with one row per source, each carrying an icon (matching the OM), the object name, and a Mode control (Union / Subtract / Intersect).

Adding sources — two paths:

  • Parent under Cubify. Drop the polygon object as a child of the Cubify in the Object Manager. It auto-appears in the Sources list and the source mesh hides itself in the viewport (the bricked output represents it now).
  • Drag-drop into the list. Drag any OM object directly onto the sources panel. The cursor changes to "accept" when valid; on drop the row appears at the bottom and the source mesh hides. Drag-dropped sources keep their original parentage in the OM — they're references rather than children.

Removing sources:

  • Click a row, then press Delete (or right-click → "Remove from list"). If the object was a direct child of the Cubify, it's also deleted from the scene; if it was drag-dropped from elsewhere, only the list link goes away and the scene object survives. Both cases unhide the source.
  • Un-parent in the OM. Dragging a child out from under the Cubify removes it from the list (and unhides it). Drag-dropped references stay even if you reparent the object elsewhere.
  • Delete the object from the scene. The list entry self-prunes on the next eval.

Multi-select: standard OS conventions — Ctrl-click to toggle individual rows, Shift-click for a range. Delete or right-click "Remove from list (N)" acts on the whole selection.

Mode column: click the Mode icon to cycle Union → Subtract → Intersect. Each mode runs as a separate voxel pass and the results are composed lattice-aligned:

  • Union — adds the source's volume to the assembly.
  • Subtract — carves the source's volume out of any Union sources.
  • Intersect — keeps only cells that are also inside this source.

Cycle protection: a Cubify cannot be a source of itself or another Cubify; an ancestor of the Cubify cannot be a source either (would freeze C4D on bake). The list rejects invalid drops with a console message.

Bind to Source Deformation

When the source is deforming (cloth dynamics, an Alembic point cache, deformer chains), this group locks each brick to the closest triangle on the source so the brick assembly rides the deformation per frame.

  • Bind Bricks to Source Deformation — toggles the binding. The bricks ride the surface in the live preview.
  • Brick Orientation — World Up keeps every brick axis-aligned; Follow Surface Normal tilts each brick to match the local surface orientation.
  • Stretch Cull Threshold — hides bricks whose anchor triangle compresses below this fraction of its rest-pose area. Useful when cloth bunches up tight enough that bricks would otherwise overlap.
  • Orientation Smoothing — dampens per-face-normal jitter on small / heavily deformed triangles. 0.7 is a good default.
  • Reference Frame — frame number used as the rest pose. 0 means "use current at fit time."
  • Re-bind to Current Frame — re-fits and re-binds with the current deformed pose as the new rest pose.

For the full simulation + render pipeline (Make Proxies → Bake to Keyframes → RBD → Swap Quality), see the deforming-mesh recipe below. For heavy deforming scenes where you want the bricks to re-flow (re-fit) every frame rather than ride a single locked layout, bake the result — see Cubify Cache below.

Cubify Cache (per-frame fit)

Bind locks one brick layout and rides the deformation. Cubify Cache does something different: it re-runs the brick fit on every frame, so the model "re-bricks itself" as it deforms — bricks reflow to suit each pose. Re-fitting live every frame would be far too slow, so you bake it once and play back the cached result.

  • Bake Fit Cache (Preview Range) — pre-computes the brick layout for every frame in the current preview range. A progress dialog shows the bake; press Esc or Cancel to stop. When it finishes, a Cubify Cache tag is automatically added to the Cubify object holding the per-frame layouts.
  • The Cubify Cache tag — while this tag is present and its Enabled toggle is on, Cubify plays back the cached per-frame reflow instead of re-fitting live (fast, smooth scrubbing and rendering). Its info field reads "Empty — run Bake Fit Cache" until you bake. Turn Enabled off, or delete the tag, to fall back to normal live behavior.
  • Persists with the scene — the cache is stored inside the .c4d file, so it survives saving and reopening the project. No need to re-bake after a reload.
  • Brick detail stays live — you can change Brick Mesh Detail (Proxy / Draft / Hero) while a cache is playing; detail is applied at draw time and does not require re-baking. Re-bake only if you change settings that affect the fit itself (brick selection, stud size, resolution, etc.).

Tip: bake at the brick detail you want to scrub with; bump detail up just for the final render. If you change which bricks are allowed or the resolution, re-bake to refresh the cache.

Tools

  • Create Proxies — bakes the live brick assembly into a static hierarchy: a Fracture object containing a per-brick Instance for every placement. This is what you reach for when you want dynamics, manual selection, or any work that needs real edit-able geometry instead of a live cache.
  • Proxy / High Res — toggles the proxy hierarchy between Proxy-quality carrier meshes (light, used during sim and interaction) and current-Brick-Mesh-Detail meshes (renderable). Quick way to hide proxies during simulation playback and bring them back for the render pass. The swap preserves per-brick logo rotation variants — 1×1s with quarter-turn variety and the 0/180 mix-flip both survive.
  • Create RS Color Material — see the Effectors section.

The Open User Manual button at the bottom of the tab opens this document.

Bricks tab

Controls which brick sizes Cubify is allowed to use, how surfaces are finished, and per-brick variation.

Choose Bricks

A thumbnail picker showing the 15 stock brick sizes (1×1 through 2×8 plus 3×3, 3×4, 3×6, 3×8). Click a thumbnail to toggle that size in/out of the active library. The current selection is the search space the fitter draws from.

Quick Select buttons:

  • All — enable every brick.
  • None — disable everything (you'll get an empty assembly until you add at least one).
  • Invert — flip the current selection.
  • 1×1 Only — useful for a "voxel-perfect" look that ignores larger bricks entirely.

Open Thumbnail Picker — pops up a larger floating picker (handy on small monitors).

Brick Selection — plates and size style

Two-column row:

  • Use Plates — when ON, the fitter can use 1-plate-tall bricks (plates) in addition to 3-plate bricks. Lets the assembly hug curves more tightly at the cost of more total pieces.
  • Brick Size Style — how aggressively Cubify picks larger bricks vs smaller ones:
    • Simple — favors larger brick sizes; fewest pieces.
    • Balanced — default mix.
    • Preserve Small Features — tries hardest to keep thin source details visible, even if it means many small bricks.

Brick Height

  • Brick Height — max height (in plates) the fitter is allowed to use. Fine / Balanced / Blocky presets below the field set the field to 2/3/6 respectively as quick shortcuts.
  • Vary Brick Heights — randomly mix tall and short bricks across the assembly for a less grid-uniform look. Variation Amount and Variation Seed below it grey out unless this toggle is on.
  • Merge Plate Stacks into Bricks — when three vertically-aligned plates can be replaced by one brick, do so. Reduces piece count.

Surface finishing & Smooth Top Surfaces

The Smooth Top Surfaces toggle replaces the lumpy top of the assembly with flat plates:

  • Cap Style:
    • Match Brick Below — cap each top column with a plate the same footprint as the brick beneath it (most uniform look).
    • Largest Merged Plates — pack the top with the biggest plates that fit (cleanest geometric look).
    • Random Library Mix — random plates from your library selection.
  • Cap Random Seed — seed for the Random Library Mix style.
  • Smooth Top Coverage — fraction of top cells that get smoothed (1.0 = full coverage; lower values leave some studs poking through).
  • Randomize Top Finish — randomizes the order in which cap plates appear during the build animation.

Per-Brick Variation

Humanize Bricks at the top gates this entire group — when off, all sub-options grey out and the assembly stays grid-perfect.

  • Humanize Bricks — when ON, every brick gets a small random offset and rotation so the assembly looks hand-built rather than CNC-perfect.
  • Brick Separation — adds a tiny gap between bricks (0–1, scene-relative). Useful for renders where you want to read the seams. 0 = bricks touch exactly.
  • Humanize Seed — controls the random pattern.
  • Position Variation — XY jitter magnitude (0–1).
  • Rotation Variation — yaw jitter magnitude (0–2). Use small values; the effect compounds across the assembly.

Look tab

Render quality, custom stud logos, preview/debug modes, live update.

Render Quality

  • Brick Mesh Detail — geometry density per brick:
    • Proxy — hollow shell + coarse studs, no tubes/ribs/indents. Lightest possible. Use for layout, animation playback, MoGraph effector setup. This is the default.
    • Draft — ~1k tris, plain studs, low detail.
    • Standard — ~5k tris, full anatomy, render-passable for distant shots.
    • Hero — ~50k verts per brick with full edge fillets, dense studs and tubes, baked stud logos. Use for hero renders.

Custom Logo

Replace the flat top-of-stud with a custom embossed logo across every brick in the assembly. Same base controls as the single Cubit object (Logo Source Mesh, Logo Rotation, Logo Fill %, Logo Height, Logo Blend, Logo Sink), plus per-brick rotation mixing for variety:

  • Mix Logo Orientation — when ON, individual bricks rotate their logos by a random amount on top of the base Logo Rotation. Off = every logo points the same way (CNC-perfect). On = looks like a hand-built model where bricks were placed at slightly different orientations.
  • Mix Amount % — fraction of bricks that get a non-base rotation. 0 = all bricks at the base rotation; 100 = every brick gets rotated.
  • Mix Seed — controls the random pattern.

Rotation choices depend on brick footprint:

  • 1×1 bricks — pick from four quarter-turns (0° / 90° / 180° / 270°) since the brick footprint is square and all four orientations are valid in the grid.
  • Larger bricks (1×N, 2×N, 3×N…) — flip between 0° / 180° only. Rotating a 1×2 by 90° would change its footprint and collide with neighbors.

Stud logos are baked into the brick mesh at Draft / Standard / Hero quality. Proxy quality skips logos for speed (the proxy bricks have plain studs). When you click Proxy / High Res after a simulation, each brick's per-instance rotation is preserved across the swap so your hand-built look survives the render-quality bump.

Preview Mode

  • Source Color — the live brick assembly with per-brick color from the source mesh. This is the normal working mode.
  • Shell Wireframe — debug mode showing brick outlines on the source shell. Useful for checking voxelization results.
  • Voxel Overlay — debug mode showing raw voxel cells, color-coded by shell depth.

Live Update / Rebuild Now

  • Live Update ON (default) — the assembly rebuilds whenever a slider changes. Best for interactive scrubbing.
  • Live Update OFF — sliders only update the cached state. Press Rebuild Now to commit. Use this on heavy scenes when scrubbing feels sluggish.

Animate tab

Per-brick build animation. Bricks fall in from above, optionally bounce or tilt, and smooth-top plates can fade in last.

Core sliders

  • Build Step (Build Progress) — 0 → 1 reveals the assembly. 0 = empty, 1 = fully built. Keyframe this for a "build" animation. This single slider drives the whole build, including the smooth-top plates.

Drop physics

  • Lift Height — how far above its final position each brick starts (in scene units).
  • Stagger Amount — 0 = all bricks land simultaneously; higher values spread the landing across the timeline by Y-layer (lower bricks land first).
  • Brick Hang Time — adds a pause before the brick starts moving. Lets later bricks build anticipation.
  • Motion Style — easing curve:
    • Ease, Ease In, Ease Out, Quadratic — classic curves.
    • Spring, Bounce — bricks overshoot and settle.
    • Slam — fast drop with hard stop (default; gives the LEGO snap).
    • Custom Curve — use the Custom Motion Curve field to draw your own.
  • Settle Length — how long each brick's fly-in takes, which controls how visible the decelerating landing is. Low = quick, snappy bricks; high = each brick glides in over a longer, clearly-decelerating window so the landing is readable without lengthening the overall timeline. The stagger auto-adjusts so the build still completes at Build Step 1.

Polish

  • Scale Bricks In — bricks scale from a point up to full size as they land (rather than appearing at full size and falling).
  • Use Rotation — bricks tumble slightly while falling.
  • Tilt Amount — how strong the tumble is. Keyframeable.

Top finish

(Only matters with Smooth Top Surfaces on.) The smooth-top plates sequence automatically off the single Build Step — each plate follows the brick beneath it and settles just as that brick lands, so the finish builds as a wave up the model (low plates first, peak plates last). See Blend Top Finish / Randomize Top Finish on the Bricks tab (Smooth Top Surfaces section).

Effectors tab

Cubify accepts standard Cinema 4D MoGraph effectors and drives per-brick matrix and color from them.

Effectors list

The list at the top is an InExclude that stores which effectors apply to this Cubify. Drag effectors in manually, or rely on auto-link: when you create a new MoGraph effector while a Cubify object is selected, that new effector is automatically appended to Cubify's Effectors list. (Pre-existing effectors are not auto-added — only newly-created ones.)

Supported effectors

Plain, Random, Shader, Delay, Formula, Step, Sound, Time, Push Apart, Inheritance, Python, ReEffector, Effector Target, Weight Effector, plus anything that derives from the C4D base effector class. They all can drive:

  • Position / Rotation / Scale — per-brick transform deltas.
  • Color (Color Mode = User Defined or Field) — per-brick display color that flows into the Create RS Color Material path.
  • Visibility — toggling the Plain effector's Visibility checkbox with a connected Field hides bricks where the field touches them. Hidden bricks drop out of both viewport and Redshift draw lists, not just shrink to zero.

Per-brick color in Redshift

Effectors that set color need a material that reads it. Click Create RS Color Material (Object tab → Tools) and Cubify builds and assigns BrickIt_PerBrick_Color (the auto-generated material) — a Redshift node graph wired to the RSObjectColor user data attribute. After that, any color set by an effector (or by the source mesh sampling) shows up at render time.

If you build the material yourself, the only requirement is a Color User Data node with Attribute Name = RSObjectColor (the case-sensitive attribute string Redshift maps the per-instance color to). The "Object Color" preset works because it stores RSObjectColor under the hood.

The action buttons (Create Proxies, Proxy / High Res, Create RS Color Material) live on the Object tab; see the Tools section above. Source-mesh visibility is now managed automatically — adding an object to the Sources list hides it; removing it unhides it. There is no longer a manual "Hide Source Mesh" button.

Workflow recipes

Render a hero shot

  1. Drop a Cubify, parent your source mesh under it (or drag-drop into the Sources list).
  2. On the Object tab → Shape, Algorithm = Detailed.
  3. On the Bricks tab, pick the brick sizes you want and a sensible variation.
  4. On the Look tab, Brick Mesh Detail = Hero.
  5. Click Create RS Color Material (Object tab → Tools) so per-brick color renders.
  6. Render.

Set up a build animation

  1. With Cubify configured to your liking on the Shape and Bricks tabs:
  2. Animate tab → keyframe Build Step from 0 (start of timeline) to 1 (end of timeline). This one slider drives the whole build, including the smooth-top plates.
  3. Pick a Motion Style (Slam is the default LEGO drop). Adjust Lift Height, Stagger Amount, and Settle Length (how slow/visible each landing is) to taste.
  4. Render with Brick Mesh Detail at Hero. (For animation playback, switch to Proxy quality first.)

Field-driven color and visibility

  1. Drop a Cubify with at least one source, and a Plain effector. The Plain auto-links into Cubify's Effectors list.
  2. On the Plain effector, set Color Mode = Field and add a Field (e.g. Random Field, Spherical Field) under Falloff.
  3. Create RS Color Material (Object tab → Tools) to wire the per-brick color path.
  4. To hide bricks the field touches: enable Visibility on the Plain effector and set Field strength such that touched clones turn off. Hidden bricks fall completely out of the scene rather than scaling to zero, so a moving Field is much lighter to interact with.

Big assembly, slow viewport

  1. Brick Mesh Detail = Proxy during interaction.
  2. Turn Live Update off; press Rebuild Now when you want to see a change.
  3. Source meshes auto-hide as soon as they're added to the Sources list, so you only see the brick output by default.
  4. Switch back to Hero only for the final render.

Bricks that follow a deforming mesh + collide with each other

When the source is deforming (cloth dynamics, an Alembic point cache, deformer chains), Cubify can lock each brick to the closest triangle on the source and ride the deformation per frame. Combined with C4D's Rigid Body Dynamics, you can have the bricks track the mesh while collisions push them apart instead of overlapping.

  1. Set up the source. Apply your deformer/cloth/Alembic to the source mesh and parent it under the Cubify (or drag-drop it into the Sources list).
  2. Bind. On the Object tab → Bind to Source Deformation group, check Bind Bricks to Source Deformation. The bricks ride the surface in the live preview.
    • Brick Orientation = Follow Surface Normal tilts each brick to match the local surface orientation. Orientation Smoothing dampens per-face-normal jitter on small / heavily deformed triangles; 0.7 is a good default.
    • Stretch Cull Threshold hides bricks whose anchor triangle compresses below the threshold ratio of its rest-pose area — useful when the cloth bunches up tight enough that bricks would otherwise overlap.
    • Reference Frame picks which frame's deformed pose is treated as the rest pose for the bind. Leave at 0 to use whatever frame is current when the fit runs; set a specific frame number to lock the rest pose to that frame (changing this re-fits).
    • Re-bind to Current Frame re-fits and re-binds with the current deformed pose as the new rest pose.
  3. Make Proxies. Once you're happy with the binding, click Create Proxies (Object tab → Tools) to spawn a real proxy hierarchy under a BrickIt_ProxySim_<source> (the auto-generated Null name) Null. The proxies follow the deformation via a Cubify Follow Surface tag added to that Null automatically.
  4. Set the timeline preview range to the frames you want to bake.
  5. Bake to Keyframes. Select the Follow Surface tag and click Bake to Keyframes (Preview Range). The bake writes per-brick keyframes, converts each instance into a real polygon mesh, flattens the hierarchy under a single bricks Null, adds a Rigid Body tag to each brick with Follow Position/Rotation set high, and disables the Follow Surface tag.
  6. Run the simulation. Scrub or play — the per-brick RBD pulls each brick toward its keyframed target while collisions push neighbors apart. Cache the simulation so playback is stable.
  7. Swap to high-res for render. On the Follow Surface tag, set Quality (Draft / Standard / Hero) and click Swap Quality. Each baked brick's mesh data is replaced in-place at the chosen render fidelity. Animation tracks, RBD tags, and the dynamics cache stay intact — the swap only changes mesh data. Templates are built lazily on first click.

Notes

  • The Source Cubify must still exist when you click Swap Quality — the swap looks up the original Cubify's brick library and template builder via a hidden link on the Follow Surface tag. Don't delete the Cubify object until after your final render.
  • The bake is a one-shot per preview range. To rebake at a different range, re-do the workflow from Make Proxies.
  • Cloth + Bullet PBD doesn't read scripted matrices as Follow targets, which is why Bake to Keyframes is required — it materializes the follower's per-frame transforms into animation tracks the simulator can read.

Tips and troubleshooting

Bricks render flat black in Redshift. Wire a Redshift material with a Color User Data node, Attribute Name = RSObjectColor. The Create RS Color Material button does this for you. Don't use the literal string "Object Color" — Redshift maps it via the underlying attribute string, not the menu label.

The assembly jumps when I add a MoGraph effector. This is fixed in the current build (effectors now anchor correctly to the source mesh's world position even when Cubify sits at a different transform). If you see it, you're probably running an older build — update to the latest preview.

Animation is sluggish when an effector is active. Drop to Proxy quality on the Look tab while you scrub, switch back to Hero for the render. The effector-active fast path rebuilds carrier instances each frame; Proxy geometry makes that ~10× cheaper.

The brickified output is missing pieces in interior cavities. That's Shell mode at work — switch Build Type on the Object tab → Shape group to Solid if you want the interior filled.

Adding 1×1 bricks to fill a boundary creates noise. Turn on Clean Small Details (Object tab → Shape), or switch Brick Size Style on the Bricks tab to Simple so the fitter prefers larger pieces.

Make Editable produces selection or material chaos. The non-Make-Editable workflow (using Create RS Color Material) is the recommended path for most renders. If you do need Make Editable, Create Proxies first — that produces a clean static hierarchy organized by source-mesh region (one Null per source child, sub-Nulls per disconnected polygon island) which is much easier to select and assign materials in.
